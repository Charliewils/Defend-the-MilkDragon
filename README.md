# 保卫奶龙 (Defend the MilkDragon)

一款基于 HTML5 Canvas 的塔防游戏，采用纯前端技术栈开发，支持战役模式与无尽模式，并集成了基于神经网络的 AI 自适应难度系统。

## 技术栈

| 层级 | 技术 |
|------|------|
| 渲染 | HTML5 Canvas 2D（纯矢量绘制，无外部图片资源） |
| 语言 | 原生 JavaScript (ES Modules) |
| AI 引擎 | brain.js（前端神经网络库） |
| 样式 | CSS3 Custom Properties + 响应式布局 |
| 桌面端 | Electron 32 |
| 打包 | electron-builder（portable / NSIS 安装包） |

## 核心功能

### 1. 双游戏模式

- **战役模式**：多关卡推进，每关包含固定波次的敌人编排，通关可获得星级评价与宝石奖励
- **无尽模式**：支持多张地图选择，波次无上限，敌人血量与速度随波次递增（HP ×1.15/波，速度最高 ×2.5），挑战历史最高波次记录

### 2. 七种防御塔

| 炮台 | 定位 | 特殊效果 |
|------|------|----------|
| 普通炮台 | 基础输出 | 稳定单体伤害，性价比高 |
| 寒冰炮台 | 减速控制 | 命中减速 60%，持续 3 秒 |
| 擎天柱 | 百分比爆发 | 额外造成目标当前血量 10% 伤害 |
| 多头塔 | 多目标火力 | 同时锁定 2 个目标开火 |
| 熔岩器 | 持续灼烧 | 附加 DoT 灼烧效果 |
| 散射炮台 | 范围扫射 | 八向散射 3×3 范围伤害，可命中隐身单位 |
| 黄油炮台 | 定身控制 | 概率定身 + 范围减速 |

所有炮台支持升级（最高 3 级），升级后伤害、射程、特效均有成长。

### 3. 八种敌人类型

| 敌人 | 特点 |
|------|------|
| 普通怪 | 血量与速度均衡，成群施压 |
| 加速怪 | 半血后进入冲刺状态，大幅提速 |
| 隐身怪 | 周期性隐身，多数炮台无法锁定 |
| 霸体战士 | 护甲环 + 控制免疫 |
| 怪物小队 | 一次生成 4 单位，队长标记 |
| 裂变虫 | 多节虫身，死亡后分裂为幼体 |
| 裂变幼体 | 由裂变虫分裂产生，低血量高数量 |
| Boss | 超高血量，半血后狂暴加速 |

### 4. AI 自适应难度系统

基于 brain.js 构建前馈神经网络，实现两大核心功能：

- **动态难度调节**：采集玩家生命损失率、金币使用率、通关速度等特征向量，输出难度乘数（0.7~1.5），实时调整敌人强度
- **智能炮台推荐**：根据当前波次的敌人构成分析威胁类型，通过克制矩阵 (TOWER_COUNTER_MATRIX) 推荐最优炮台搭配，并生成自然语言提示

训练参数：迭代 2000 次，误差阈值 0.005，学习率 0.3。

### 5. 道具系统

游戏内可装备 4 种主动道具：

- **闪电风暴** — 全屏范围伤害
- **时间冻结** — 暂停所有敌人移动
- **金币雨** — 快速获取额外金币
- **强化脉冲** — 临时提升所有炮台伤害

### 6. 经济与收藏系统

- **宝石商店**：击杀 Boss、完美通关、无尽模式可获得宝石
- **外观系统**：使用宝石解锁萝卜皮肤和炮台外观
- **图鉴系统**：收录所有敌人与炮台的详细数据、克制关系

### 7. 存档系统

基于 localStorage 实现多槽位存档：
- 战役进度独立保存
- 无尽模式历史记录
- 中途离开自动保存游戏状态，返回后可继续

## 运行方式

### 浏览器直接运行

使用任意本地 HTTP 服务器托管项目目录：

```bash
# 方式一：使用自带的 Python 服务器
python3 serve.py

# 方式二：使用 Python http.server
python3 -m http.server 8000

# 方式三：使用 Node.js 的 npx
npx serve .
```

然后在浏览器中访问 `http://localhost:8000`。

### Electron 桌面客户端

```bash
# 安装依赖
npm install

# 启动开发模式
npm start

# 打包 Windows 可执行文件（portable + NSIS 安装包）
npm run dist
```

## 设计亮点

1. **纯矢量渲染**：所有游戏元素均通过 Canvas API 程序化绘制，无需加载外部图片资源，首屏加载快
2. **模块化架构**：采用 ES Modules 组织代码，实体继承体系清晰（Tower → 各塔子类，Enemy → 各敌人子类）
3. **AI 驱动体验**：神经网络实时分析玩家行为并调整难度，避免过易或过难
4. **完整的游戏循环**：包含经济系统、升级系统、存档系统、成就收藏等完整游戏设计
